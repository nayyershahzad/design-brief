# sample-next-app

A minimal Next.js (App Router) project used to prove that a **design-brief export drops into a real app cleanly**. It is intentionally *not* part of the monorepo's npm workspaces or build — it's a standalone target.

## What's here

```
app/
  tailwind.css     # hand-owned: @tailwind base/components/utilities — design-brief NEVER touches this
  globals.css      # machine-owned: the shadcn :root token block, generated by design-brief
  layout.tsx       # imports tailwind.css then globals.css (order matters)
  page.tsx         # a page styled entirely from the tokens
direction.json         # the locked Direction this export came from (preset: clean-teal)
design-brief.theme.json # theme.extend additions, merged in tailwind.config.ts
DESIGN_SPEC.md          # the human-readable design contract
tailwind.config.ts      # maps shadcn vars + spreads design-brief.theme.json
```

The key idea: **design-brief owns `app/globals.css` and the two root artifacts, and nothing else.** Tailwind's own directives live in `app/tailwind.css`, so design-brief can regenerate `globals.css` on every lock without ever clobbering your hand-written setup. That's what makes "machine-owned, regenerate freely" true.

## Regenerate the export

The real flow is the playground. From this directory, with the CLI built (`npm run build` at the repo root):

```bash
node ../../packages/cli/dist/bin.js play
```

design-brief detects this project, finds the existing `app/globals.css`, and on **lock** writes the three artifacts back in place: `app/globals.css`, `design-brief.theme.json`, and `DESIGN_SPEC.md`. Because export is deterministic, locking the same direction reports `unchanged`; `app/tailwind.css` is never touched. Lock a different direction and the whole app restyles — no component edits.

There's also a headless export for scripting, which writes the bundle (`globals.css`, `design-brief.theme.json`, `DESIGN_SPEC.md`) flat into a directory you choose:

```bash
node ../../packages/cli/dist/bin.js export direction.json -o ./out
```

## Run it (optional)

```bash
npm install
npm run dev
```

Everything on the page — background, card, primary button, muted text — resolves from the variables in `app/globals.css`.
